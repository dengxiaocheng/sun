# 02｜接入现有 sun 玩法

## 现有 sun 核心状态

根据现有 `sun` README，游戏主界面统一展示 7 条中文进度条：

| 缩写 | 中文标签 | 支线中的作用 |
|---|---|---|
| A | 焦虑雾 | 延误、拥挤、不回消息、低电量都会提高。 |
| M | 动能 | 每完成一个交通段都会提高，体现“我真的能走完”。 |
| P | 复习进度 | 旅途中可小幅推进，但不能鼓励过度硬学。 |
| R | 恢复值 | 长途疲惫核心消耗项。吃饭、闭眼、睡觉可恢复。 |
| B | 边界感 | 是否要求接站、休息、不要立刻社交。 |
| T | 亲密温度 | 邓孝程是否回应具体、是否接站、是否照顾。 |
| E | 理解/共情 | 邓孝程是否理解这趟路对孙铭欣意味着什么。 |

本支线不替换主数值系统，只新增一组临时旅途状态，在章节结束时折算回主数值。

## 新增临时状态

| 状态 | 范围 | 说明 |
|---|---:|---|
| 旅程余裕 U | 0-100 | 时间缓冲。打车堵车、安检排队、迷路会降低。 |
| 体力 F | 0-100 | 旅途核心资源。连续移动会消耗，休息和进食可恢复。 |
| 手机电量 C | 0-100 | 影响导航、联系邓孝程、查车次。 |
| 行李负担 L | 0-100 | 越重越消耗体力。行前打包决定。 |
| 连接感 K | 0-100 | 孙铭欣是否觉得不是自己一个人在硬撑。 |

## 接入方式

在主线场景管理中新增一个章节类型：`travelChapter`。该章节仍使用原有对话框、选择按钮、状态条反馈，但顶部或侧边临时显示：

- 旅程余裕
- 体力
- 手机电量
- 连接感

行李负担不必常驻显示，可在行前打包界面和体力结算时出现。

## 章节结算

旅程结束后临时状态折算为主状态：

| 条件 | 主状态影响 |
|---|---|
| U ≥ 60 | A -5，M +5 |
| F ≥ 50 | R +10 |
| F < 25 | R -15，A +10 |
| C < 15 | A +5 |
| K ≥ 70 | T +15，E +10 |
| K < 30 | T -10，A +10 |
| B ≥ 60 且邓孝程接站良好 | B +10，T +10 |
| 完成 T00-T08 全流程 | M +15 |

## 玩法目标

玩家不是要把焦虑压到 0，而是让孙铭欣在焦虑中继续完成下一段路。最好的体验是：

1. 她知道自己累。
2. 她会主动照顾身体。
3. 她会提出具体需求。
4. 邓孝程开始用具体行动回应。
